Detailed Description

template<typename N, typename S>
class qi::sock::ConnectSocketFuture< N, S >

Connects to a URL and gives the created socket via a future.

The only difference relatively to ConnectSocket is that the connected socket is presented via a future instead of a handler.

Usage:

* ConnectSocketFuture<N> connect{ioService};
* connect(Url{"tcp://1.2.3.4:9876"}, SslEnabled{false}, IpV6Enabled{false});
* auto socketPtr = connect.complete().value();
*

Warning: in the same manner as ConnectSocket, the ConnectSocketFuture object must remain alive until the connecting process is complete.

Remark: See ConnectSocket for an explanation on the stop.

Network N, With NetSslSocket S: S is compatible with N
