Detailed Description

template<typename N, typename S>
struct qi::sock::Connected< N, S >

Connected state of the socket. Allow to send and receive messages.

Received messages are exposed via a callback specified at construction.

You can send messages continuously : they will be enqueued if needed. By default, enqueued messages are sent as soon as possible, but you can explicitly ask to stop the queue processing.

You can request this state to stop, by optionally passing it a Promise to be set on disconnection.

A complete future is set when the connected state is over. It contains the socket and a Promise to be set when disconnection is over.

If an error occurs while sending or receiving, the complete future is set in error.

Warning: You must call stop() before closing the socket, because closing the socket cause the send and receive handlers to be called with an 'abort' error, and we don't want to take it into account.

Usage:

* auto onReceive = [](ErrorCode e, const Message* msg) {
* // `msg` is valid only if there is no error.
* // Do something with `msg`.
* return true; // Or return false to stop listening to message.
* };
* Connected<N> c{socketPtr, SslEnabled{true}, maxPayload, onReceive};
* // Get a Message msg;
* c.send(msg); // Move the message if you can, to avoid a copy.
* // Send more messages.
* Future<std::pair<SocketPtr<S>, Promise<void>>> futureComplete = c.complete();
*
